# Build a Figma Featured In Press Carousel on Horizon Slideshow Primitives

## Problem

The Ogee homepage redesign required a Featured In carousel matching Figma
desktop and mobile frames while remaining editable in Shopify's theme editor.
Desktop uses a two-column press slide with a 320px image lane, a 760px quote
panel, arrow controls under the quote, and a centered All Press button. Mobile
removes the image lane, keeps a 360px press card, changes navigation to a
thin segmented progress line, and makes the All Press button full width.

The theme already had Horizon slideshow primitives and custom homepage
sections. The new work needed to avoid adding another carousel runtime or
overloading shared product/hero carousel files.

## Symptoms

- A generic card carousel does not match the press layout because the desktop
  image column is present only above the mobile breakpoint.
- Figma uses arrows on desktop but a 1px progress indicator on mobile.
- Quote copy needs inline formatting, such as an underlined product name,
  without turning the quote into a non-editable image.
- The supplied Figma asset URLs are temporary, and uploaded clipboard images
  live outside the repository.
- Exact font matching depends on Helvetica Neue and Chronicle Display being
  available to the storefront.
- `templates/index.json` is generated by Shopify and can be overwritten by the
  theme editor, but it is still the local route-to-section map for this branch.

## What Didn't Work

### Extending the product carousel

The product carousel owns tabs, collection data, product-card rendering, and
CTA updates. Adding press slides there would mix unrelated data contracts and
make future product carousel changes riskier.

### Using a section-local block loop only

Looping directly over `section.blocks` can produce static markup, but it would
miss the theme's existing `slideshow-component` behavior for swipe, keyboard
selection, selected-state updates, and disabled arrow handling.

### Flattening the desktop design into one image

A flattened press card would preserve pixels but remove merchant-editable text,
harm accessibility, and make the mobile variant harder to maintain.

### Relying on temporary Figma asset URLs

Figma MCP assets expire. For a branch that should render later, move the
provided reference imagery into theme assets or use merchant-selected
`image_picker` values rendered through Shopify.

### Trusting theme font defaults

The local theme settings currently point to Inter. The Figma references use
Helvetica Neue and Chronicle Display, so the section needs explicit font
stacks for the press component while still accepting graceful fallbacks.

## Solution

Create a dedicated section and a dedicated theme block:

- `sections/featured-in-carousel.liquid`
- `blocks/_featured-in-press-slide.liquid`

Reuse Horizon slideshow primitives:

- `snippets/slideshow.liquid`
- `snippets/slideshow-slide.liquid`
- `snippets/slideshow-controls.liquid`
- `assets/slideshow.js`

The section owns:

- Section padding, content width, colors, and button styling.
- Shared eyebrow text.
- The All Press CTA.
- Slideshow assembly with desktop arrows and mobile progress styling.
- Empty state copy when no press slides are configured.

The block owns:

- One press mention.
- Merchant image override through `image_picker`.
- Bundled image fallback selection for the provided Vogue and Saatva imagery.
- Publication logo text, optional subtext, logo style, and rich quote.
- `block.shopify_attributes` through the rendered slideshow slide.

Wire the homepage JSON:

- Add a `featured_in_carousel` section to `templates/index.json`.
- Add Vogue and Saatva press blocks using the bundled asset fallbacks.
- Append the new section after the existing promotion banner in template order.

Use `inline_richtext` for quotes so inline emphasis can be edited without
adding wrapping paragraphs. Use CSS custom properties for merchant-controlled
single values like colors, padding, and content width. Keep structural layout
differences in responsive CSS.

Use explicit component font stacks:

- Helvetica Neue stack for eyebrow, button, mobile quote, and Saatva-style logo.
- Chronicle Display stack for the desktop quote, with the heading font as a
  fallback.
- A serif approximation for the Vogue-style text logo when no logo image is
  supplied.

## Why This Works

The section reuses the existing carousel runtime, so swipe, arrows, dot
selection, `aria-hidden` state, and viewport-aware scrolling stay consistent
with the rest of the Horizon theme.

The dedicated block isolates the Figma-specific press composition from product
and hero sections. Merchants can reorder, duplicate, or replace press mentions
without editing Liquid.

The bundled asset setting lets the provided images render immediately on the
branch while still giving merchants a proper `image_picker` override. This is
safer than relying on temporary Figma URLs and more flexible than hardcoding a
single image.

The mobile and desktop controls share one `slideshow-controls` render, then
CSS presents arrows on desktop and segmented progress on mobile. That avoids a
second control implementation while matching the two Figma treatments.

## Prevention

- Inspect existing carousel primitives before adding JavaScript.
- Keep Figma-specific composition in a narrow section/block pair rather than
  expanding shared carousel files.
- Include `block.shopify_attributes` on rendered slide wrappers for theme
  editor selection.
- Use `inline_richtext` when quotes need inline formatting but not paragraph
  wrappers.
- Move provided local reference images into versioned theme assets if they need
  to render without merchant uploads.
- Do not rely on temporary Figma asset URLs for production theme code.
- Keep visible 1px progress indicators backed by at least a 44px touch target.
- Validate section and theme-block schema after edits.
- Verify desktop, tablet, and mobile in a Shopify preview because exact font
  availability and rendered heights cannot be fully proven from schema checks.
- Treat `templates/index.json` edits as launch wiring that may later be
  overwritten by Shopify admin changes.
- Confirm the All Press URL exists before launch.

## Reusable Insight

For editorial or press carousels from Figma, separate carousel behavior from
composition. Reuse the theme's existing slideshow runtime, but create a
dedicated block for the content model and responsive art direction. This keeps
the interactive contract stable while giving the Figma-specific layout enough
room to be precise.

## Compound Summary

The Featured In carousel was implemented as a reusable Shopify section and
theme block on top of Horizon slideshow primitives. It uses bundled press image
fallbacks from the provided assets, editable quote/logo settings, desktop
arrow controls, mobile segmented progress, and homepage JSON wiring.

Review found no blocking issues after validation. One design-fidelity issue was
corrected during review: component typography now uses explicit Helvetica Neue
and Chronicle Display stacks instead of relying only on the theme's current
Inter settings. Shopify Liquid validation passed for the new section, new block,
and homepage template using the local schema cache. Remaining launch checks are
visual Shopify preview at desktop/tablet/mobile sizes, real font availability,
and confirming that `/pages/press` exists.
